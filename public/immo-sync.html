BUT DU PROJET

Ce projet vise à développer une application dédiée à la gestion et au suivi de travaux. Elle permet de centraliser les projets, de suivre l'avancement des chantiers et de faciliter la coordination entre les différents utilisateurs — propriétaires, inspecteurs et entrepreneurs.

Chaque utilisateur dispose d'un espace adapté à son rôle. Le projet comprend également un espace administrateur et un espace gestionnaire, offrant un contrôle global sur l'application, la gestion des utilisateurs, des projets et des données associées.

DIFFÉRENTES VERSIONS

V1 — Java

Gestion globale pour le gestionnaire : entrepreneurs, catégories, propriétaires, biens, prestations. Base fonctionnelle de l'application.

V2 — Java / Web Mobile

Création et gestion des chantiers après inspection. Côté inspecteur : interface web mobile pour consulter les chantiers, localiser les biens sur carte et saisir des observations.

V3 — Java

Sollicitation automatisée des entrepreneurs après validation du chantier, avec algorithmes de calcul de distances géographiques pour cibler les professionnels proches.

V4 — Web Symfony

Espace entrepreneur dédié en Symfony : consultation des demandes de chantiers, accès aux devis types, soumission de propositions via formulaires sécurisés.

V5 — Java

Outil de comparaison des devis avec interface graphique enrichie, jeu de couleurs pour la lisibilité, et algorithmes multi-critères pondérés (date + coût).

DIFFICULTÉS RENCONTRÉES

Gestion du projet avec GitHub

La prise en main de GitHub a nécessité une adaptation, notamment pour organiser le dépôt, gérer les branches et assurer un suivi clair des versions du projet.

Gestion des rôles et des droits utilisateurs

La mise en place de profils distincts (gestionnaire, inspecteur, entrepreneur, propriétaire) a impliqué une réflexion approfondie sur les autorisations et les accès selon chaque rôle.

Conception et structuration de la base de données

La modélisation des données a représenté un enjeu important pour garantir la cohérence entre les entités et assurer de bonnes performances lors des requêtes.

Organisation et évolution progressive des fonctionnalités

Le projet a été développé par versions successives, ce qui demandait une bonne anticipation afin d'éviter les régressions lors de l'ajout de nouvelles fonctionnalités.

CONCLUSION

Ce projet de gestion de chantiers m'a permis de développer une application complète en Java et web, intégrant plusieurs profils utilisateurs et des fonctionnalités évolutives. Le travail par versions successives m'a aidé à mieux comprendre l'architecture d'une application, la structuration des données et la gestion d'un projet informatique.

Les difficultés rencontrées — GitHub, coordination des technologies, conception de la base de données — ont renforcé ma rigueur et mon autonomie. Ce projet m'a permis de consolider mes compétences techniques et de mieux me préparer aux exigences du monde professionnel.
